# Awesome-lists & directories

Highest ROI / lowest effort — do these first. Most "awesome" lists take a one-line entry; some require a specific submission flow.

## Submission blurb (reuse everywhere — keep `npx claudescope-cli` verbatim)

**Short (one-liner):**
```
ClaudeScope — `npx claudescope-cli`. Local-first, zero-dependency dashboard + full-text search over your Claude Code session transcripts. Search every prompt, reply and tool call you've ever sent and jump into the session; plus token/cost stats, per-project & per-model spend, an activity heatmap, and an anonymized share card. 100% local, no network. MIT.
```

**With link:**
```
[ClaudeScope](https://github.com/JoniMartin27/claudescope) — Local-first dashboard + full-text search & replay for your Claude Code history (`npx claudescope-cli`). Tokens/cost stats, per-model/project spend, heatmap, anonymized Wrapped card, installable PWA. Zero deps, 100% local, MIT.
```

## Where to submit

- [ ] **hesreallyhim/awesome-claude-code** — ⚠️ submit via the repo's **issue form** (`submit-resource.yml`), NOT a pull request (PRs are closed). Category: Tooling / Applications. Use the short blurb.
- [ ] **jqueryscript/awesome-claude-code** — PR a one-line entry under tools/applications.
- [ ] **rohitg00/awesome-claude-code-toolkit** — companion-apps / tools section.
- [ ] **awesomeclaude.ai** — submit via their site form if present.
- [ ] **claudefa.st** directory — submit listing.
- [ ] **npm keywords** are already set (`claude-code`, `search`, `session-search`, `local-first`, `privacy`…) so it shows up in npm search for those.

## Tips
- Lead the entry with **search**, not analytics — that's the differentiated keyword space.
- Pair the listing with the repo URL + `npx claudescope-cli` so word-of-mouth doesn't 404 on the bare name.
- After 0.x gains a little traction, re-submit to each list's "what's new" with the Wrapped card as the hook.
